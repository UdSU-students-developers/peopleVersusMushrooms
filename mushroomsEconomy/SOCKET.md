## 1. Общее

### 1.1. Адрес сервера

`ws://localhost:3005/ws`

### 1.2. Протокол подключения

Подключение происходит по протоколу WebSocket. После установки соединения клиент должен авторизоваться, отправив событие `auth` с токеном.

Формат всех сообщений — `JSON`.

## 2. Структуры данных

### 2.1. Общий формат сообщения

```
SocketMessage<T>: {
    event: string;    // название события
    data?: T;         // данные события
    error?: {         // ошибка (если есть)
        code: number;
        message: string;
    };
}
```

### 2.2. Пользователь

```
User: {
    id: number;
    token: string;
    name: string;
    guid: string;
    passwordHash: string;
    token: string;
}
```

## 3. События (Server → Client)

| Событие | О чем                                        |
| -------------- | ------------------------------------------------ |
| MESSAGE        | Сообщение пришло                  |
| REGISTRATION   | Ответ на регистрацию           |
| LOGIN          | Ответ на авторизацию           |
| LOGOUT         | Ответ на выход из аккаунта |

### 3.1. Общие ошибки

* `11` - Ошибка авторизации
* `12` - Mushroom с данным id не найден
* `13` - Передан неполный набор параметров
* `14` - Unit с данным id не найден
* `15` - Некорректная матрица
* `16` - Пользователя с данным именем не существует
* `17` - Пользователь с данным именем уже зарегистрирован

## 5. Подробно (Server → Client)

### 5.1. MESSAGE

Пришло сообщение

**Данные при успехе**

```
{
   name: string;
   text: string;
}
```

### 5.2. REGISTRATION

Ответ на запрос регистрации.

**Данные при успехе**

**json**

```
{
    "id": 42,
    "guid": "uuid-пользователя",
    "name": "Игрок",
    "token": "сгенерированный-токен"
}
```


**Возможные ошибки**

* `13` – не переданы `name` или `password`
* `17` – пользователь с таким именем уже существует

### 5.3. LOGIN

Ответ на запрос авторизации.

**Данные при успехе**
Аналогично регистрации: возвращается объект пользователя с полями `id`, `guid`, `name`, `token` и др.

**Возможные ошибки**

* `13` – не переданы `name` или `password`
* `11` – неверное имя пользователя или пароль

### 5.4. LOGOUT

Ответ на запрос выхода из аккаунта.

**Данные при успехе**

**json**

```
true
```

(просто булево значение, подтверждающее выход)

**Возможные ошибки**

* `13` – не передан `token` или `guid`

Отправка сообщения

```

   name: string;
   text: string;

```


### 6.2. REGISTRATION

Запрос на создание нового пользователя.

```
    "password": "пароль"

```

---

### 6.3. LOGIN

Запрос на авторизацию существующего пользователя.

**Данные**

**json**

```
{
    "name": "Имя игрока",
    "password": "пароль"
}
```

---

### 6.4. LOGOUT

Запрос на выход из аккаунта (завершение сессии).

**Данные**

**json**

```
{
    "token": "токен-пользователя",
    "guid": "идентификатор-пользователя"
}
```
