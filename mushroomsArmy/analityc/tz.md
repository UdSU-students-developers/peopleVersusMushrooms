# АВТОРИЗАЦИЯ ПОЛЬЗОВАТЕЛЯ
1. Общие требования
Разработать систему авторизации пользователей в игру. Интерфейс представляет собой страницу входа: заголовок «Вход», два поля ввода (логин и пароль), кнопка «Войти» и ссылка для перехода к регистрации.
После успешного входа пользователь автоматически перенаправляется в лобби — центральное пространство для подготовки к игре.

2. Функциональные требования
    2.1. Страница входа
        2.1.1. Элементы интерфейса:
        * Заголовок страницы: «Вход»
        * Поле ввода логина с placeholder «Введите логин»
        * Поле ввода пароля с placeholder «Введите пароль» (тип input: password)
        * Кнопка «Войти» (активна при заполнении обоих полей)
        * Ссылка «Нет аккаунта? Зарегистрироваться» (переход на страницу регистрации)

        2.1.2. Входные данные:
        * Логин — строка от 3 до 20 символов. Допустимы латинские буквы, цифры, символы подчёркивания и точки. Логин не может начинаться или заканчиваться точкой, не может содержать две точки подряд.
        * Пароль — строка от 6 до 50 символов. Допустимы любые символы.

        2.1.3. Процесс авторизации:
        * Пользователь вводит логин и пароль
        * Нажимает кнопку «Войти»
        * Система проверяет существование пользователя с указанным логином
        * Если пользователь найден, система сравнивает введённый пароль с сохранённым хешем
        * При успешной проверке система генерирует сессионный токен
        
        2.1.4. Валидация:
        * Оба поля обязательны для заполнения
        * Кнопка «Войти» неактивна, пока поля пустые
        * При вводе некорректных данных показывается сообщение об ошибке
        
        2.1.5. Результат:
        * При успешном входе: возвращается сессионный токен, информация о пользователе, автоматический переход в лобби
        * При ошибке: показывается сообщение «Неверный логин или пароль»

    2.2. Регистрация нового пользователя
        2.2.1. Переход к регистрации:
        * Пользователь нажимает на ссылку «Нет аккаунта? Зарегистрироваться» на странице входа
        * Открывается страница регистрации (отдельный экран или модальное окно)

        2.2.2. Входные данные:
        * Логин — строка от 3 до 20 символов. Допустимы латинские буквы, цифры, символы подчёркивания и точки. Логин не может начинаться или заканчиваться точкой, не может содержать две точки подряд.
        * Пароль — строка от 6 до 50 символов. Допустимы любые символы.
        * Подтверждение пароля — повторный ввод пароля для проверки

        2.2.3. Валидация:
        * Логин должен быть уникальным в системе
        * Пароль не должен совпадать с логином
        * Пароль и подтверждение пароля должны совпадать
        * Все поля обязательны для заполнения

        2.2.4. Результат:
        * При успешной регистрации: создаётся запись в базе данных, пользователю возвращается статус успеха, автоматический вход в систему и переход в лобби
        * При ошибке: показывается сообщение с указанием причины:
            - «Логин уже занят»
            - «Логин не соответствует формату»
            - «Пароль слишком короткий» (менее 6 символов)
            - «Пароль слишком длинный» (более 50 символов)
            - «Пароли не совпадают»    

3. Сценарии использования
    3.1. Успешный вход
    * Пользователь открывает страницу входа
    * Вводит корректный логин
    * Вводит корректный пароль
    * Нажимает кнопку «Войти»
    * Система проверяет данные
    * Пользователь перенаправляется в лобби

    3.2. Ошибка при входе
    * Пользователь вводит неверный логин или пароль
    * Нажимает «Войти»
    * Показывается сообщение «Неверный логин или пароль»
    * Пользователь остаётся на странице входа

    3.3. Переход к регистрации
    * Пользователь нажимает «Нет аккаунта? Зарегистрироваться»
    * Открывается страница регистрации
    * После успешной регистрации — автоматический вход и переход в лобби

    3.4. Валидация полей
    * Пользователь начинает вводить данные
    * Кнопка «Войти» остаётся неактивной, пока поля не заполнены
    * При вводе некорректного формата — показывается подсказка

# ЛОББИ
1. Общие требования
Разработать экран лобби — центральное пространство после успешной авторизации. Предназначен для запуска игры и выхода из аккаунта.

2. Функциональные требования
    2.1. Элементы интерфейса
    * Кнопка «Начать игру» — расположена по центру экрана
    * Кнопка «Logout» — расположена в правом верхнем углу

    2.2. Процесс запуска игры
    * Пользователь нажимает кнопку «Начать игру»
    * Система выполняет переход на экран «Игра»
    * Сессия пользователя сохраняется

    2.3. Процесс выхода
    * Пользователь нажимает кнопку «Logout»
    * Система удаляет сессионные данные (токен, кэш)
    * Выполняется переход на экран авторизации

3. Сценарии использования

    3.1. Запуск игры
    * Пользователь находится в лобби
    * Нажимает «Начать игру»
    * Открывается экран игры

    3.2. Выход из аккаунта
    * Пользователь нажимает «Logout»
    * Сессия завершается
    * Пользователь перенаправляется на страницу входа

    3.3. Повторный вход
    * После завершения игры пользователь возвращается в лобби
    * Состояние экрана сбрасывается к исходному

# ЮНИТЫ КЛАССОВ
1. СПОРОМЕТ
    1.1.  Общее описание
    Юнит дальнего боя. Роль — поддержка и добивание врагов с безопасной дистанции.

    1.2. Характеристики
    Здоровье — 8. Скорость — 1 м/с. Радиус атаки — 12 м. Дистанция отступления — 8 м. Перезарядка — 2 с. Прицеливание — 0.5 с.

    1.3. Атака
    Наносит 5 единиц физического урона.

    1.4. Поведение
    При дистанции до врага менее 10 м — отступает. При дистанции 8–12 м — атакует. При дистанции более 12 м — двигается к врагу. При отсутствии врагов двигается к центру карты.

    1.5. Взаимодействие
    Если путь перекрыт другим юнитом, Споромет ждёт 0.5 секунды, затем пересчитывает маршрут.

2. ШАМПИНЬЕБ
    2.1. Общее описание
    Юнит ближнего боя. Роль — прорыв к вражеским рядам и уничтожение ключевых целей ценой собственной жизни.

    2.2. Характеристики
    Здоровье — 35. Скорость — 3 м/с. Радиус взрыва — 6 м. Урон взрыва — 60.

    2.3. Атака
    Взрывается при контакте с врагом или при смерти.

    2.4. Поведение
    Шампиньеб находит ближайшего видимого врага. Если враг не найден — двигается к центру карты. Если враг найден — двигается к нему. При достижении дистанции менее 6 м взрывается, нанося 60 урона всем врагам в радиусе. Не отступает. При получении урона не взрывается (только при контакте или смерти).

    2.5. Взаимодействие
    Если путь перекрыт другим юнитом, Шампиньеб ждёт 0.5 секунды, затем пересчитывает маршрут.

3. ЕБЛЕКАРЬ
    3.1. Общее описание
    Юнит поддержки. Роль — лечение союзных грибов и восстановление их боеспособности.

    3.2. Характеристики
    Здоровье — 40. Скорость — 1 м/с. Радиус лечения — 10 м. Время перезарядки лечения — 3 секунды. Количество лечения за один раз — 15 HP.

    3.3. Атака/Лечение
    Еблекарь не атакует врагов. При выборе цели он прицеливается в течение 0.5 секунды, после чего выпускает снаряд. Снаряд летит по прямой к союзному юниту. При попадании восстанавливает 15 единиц здоровья. Лечение не работает на Взрывоморы и Споровые башни — только на подвижные юниты (Споромет, Шампиньеб, другие Лекари).

    3.4. Поведение
    Цикл принятия решений выполняется каждые 0.5 секунды. Еблекарь находит ближайшего видимого союзного юнита, у которого здоровье ниже максимального. Если такой юнит найден и расстояние до него менее 10 метров — Еблекарь останавливается и лечит. Если расстояние более 10 метров — двигается к нему. Если все союзники здоровы или союзников нет — Еблекарь остаётся на месте. При получении урона не меняет поведение, продолжает лечить союзников.

# ПРОИЗВОДСТВО ЮНИТОВ
Экономика грибов управляет созданием юнитов в инкубаторе. Когда юнит создан, система отправляет событие с параметром тип юнита (Споромет, Шампиньеб, Еблекарь). Получив событие, мы отправляем запрос на карту по guid для размещения юнита. После размещения на карте активируется его поведение.

# ПЕРЕДВИЖЕНИЕ ПО КАРТЕ
1. Общее описание
Реализовать перемещение юнитов (Споромет и Шампиньеб) по карте. Для расчёта маршрута используется библиотека easystar.js. Карта предоставляется сервисом карт.

2. Расчёт пути
Получить от сервиса карт двухмерный массив проходимости клеток. Настроить easystar.js на работу с этим массивом.

Для каждого юнита определять целевую позицию и строить путь от текущих координат до целевых. Если путь существует, юнит последовательно перемещается по точкам пути. Если путь не найден, юнит остаётся на месте до следующего цикла.

3. Поведение при движении
    3.1. Споромет:
    При дистанции до врага менее 8 м — отступает. При дистанции 8–12 м — атакует. При дистанции более 12 м — двигается к врагу.

    3.2. Шампиньеб:
    Двигается к ближайшему врагу. При дистанции менее 6 м — взрывается.
    
    3.3. Еблекарь:
    Двигается к ближайшему раненому союзнику. При дистанции менее 10 м — лечит.

4. Обнаружение цели
Каждые 0.5 секунды юнит ищет ближайшего видимого игрока - врага. Юнит считается видящим врага, если между ними нет непроходимых препятствий (стен, закрытых зон) согласно карте проходимости. Если враг не найден, юнит двигается к центру карты.

# ОБОРОНИТЕЛЬНЫЕ СООРУЖЕНИЯ
1. Общее описание
Грибы возводят на карте оборонительные сооружения, чтобы замедлить продвижение людей и защитить ключевые точки. Сооружения размещаются на карте, имеют прочность и могут быть разрушены людьми.

2. Взрывомор
    2.1. Характеристики
    Прочность — 70. Радиус срабатывания — 7 м. Урон — 35. Занимает 1 клетку. Непроходим для людей. Грибы проходят сквозь.

    2.2. Поведение
    При попадании человека в радиус срабатывания взрывается, наносит 35 урона, исчезает. Если взрыв убил человека — восстанавливается через 5 секунд. При уничтожении выстрелом — не восстанавливается.

    2.3. Отображение
    Занимает всю клетку. При прочности менее 50% отображается в повреждённом состоянии.

3. Споровая башня
    3.1. Характеристики
    Прочность — 160. Занимает 2x2 клетки. Радиус атаки — 20 м. Перезарядка — 2 с. Урон — 15. 

    3.2. Поведение
    Споровая башня автоматически атакует ближайшего видимого человека, находящегося в радиусе атаки. При отсутствии целей находится в режиме ожидания. Грибы не получают урон от башни. Башня является непроходимой для людей и для грибов.

    3.3. Отображение
    Крупный объект, занимающий четыре клетки. При прочности менее 50% отображается в повреждённом состоянии.

4. Взаимодействие сооружений
На этапе генерации карты Взрывоморы выстраиваются в стены длиной до 8 клеток. Споровые башни располагаются за ними.
