Сущности

1.Термины и определения

Игрок — пользователь, вошедший в игру.

Юнит — управляемая игровая единица.

Спавн — место появления игрока на карте.

Ресурсы — различные игровые валюты.

Скорость роста — скорость роста травы на мицелии.

PvP — режим «игрок против игрока».

HP (Health Points) — показатель здоровья юнита, определяющий, сколько урона он может выдержать перед уничтожением.

DPS (Damage Per Second) — показатель урона в секунду, который юнит наносит атаками.

2.Цели и задачи
Цель

Разработать экономическую систему PvP-стратегии с элементами строительства базы.

Задачи

Реализовать клиентскую часть экономики.

Реализовать серверную часть экономики.

Создать сбалансированную систему строительства и развития базы.

Обеспечить стабильную работу мультиплеера.

Провести тестирование работы приложения.
Вход в игру 
Состоит из двух полей ввода:
1. Логин
2. Пароль(при вводе зашифровано точками)
3. чекбокс "запомнить меня"

И кнопок:
1. "Авторизоваться"
2. "зарегистрироваться"

### 3.2 Экран регистрации  
1. **Поле ввода "Логин"**
   - Валидация в реальном времени
   - Индикатор доступности логина

2. **Поле ввода "пароль"**  
   - при вводе зашифровано точками

3. **Поле ввода "Подтверждение пароля"**
   - Сравнение с исходным пароле
   - Визуальное подтверждение совпадения

4.   **Поле ввода "отображаемое имя"**

4. **Кнопка "Зарегистрироваться"**
   - Активируется только при выполнении всех требований

### требования логина и пароля
**Требования к логину:**
- Длина: от 5 до 20 символов
- Допустимые символы: латинские буквы (a-z, A-Z), цифры (0-9), символы подчеркивания (_)
- Не должен начинаться с цифры или символа подчеркивания
- Не должен содержать пробелов и специальных символов (@, #, $, %)
- Должен быть уникальным (проверка на занятость в реальном времени)
- не должен представлять из себя персональную информацию  

**Требования к паролю:**
- Минимальная длина: 8 символов
- Обязательное использование символов разных регистров (A-Z, a-z)
- Обязательное наличие хотя бы одной цифры (0-9)
- Рекомендуется использование специальных символов (!@#$%^&*)
- пароль не должен совпадать с логином 

**Дополнительные элементы:**
   - Ссылка "Забыли пароль?" для восстановления доступа
   - Ссылка "Создать аккаунт" для перехода к регистрации
   - Чекбокс "Запомнить меня" для сохранения сессии

   **забыли пароль:**
    - просит ввести логин

# UserManager

## Структура и назначение

UserManager расширяет BaseManager и управляет пользователями, их регистрацией, входом/выходом из системы.

## Основные компоненты

### 1. Конструктор

- Инициализирует хранилище пользователей `users` (ключ - guid, значение - экземпляр User)
- Настраивает Socket.IO обработчики событий:
  - `REGISTRATION` - регистрация
  - `LOGIN` - вход
  - `LOGOUT` - выход
  - `disconnect` - отключение сокета
- Настраивает mediator для получения пользователя по guid

### 2. Хранилище пользователей

Каждый пользователь хранится как экземпляр класса User.

### 3. Методы

| Метод | Описание |
| :--- | :--- |
| `triggerGetUserByGuid(guid)` | Возвращает пользователя по guid, если он существует и авторизован |
| `socketRegistration(data, socket)` | Обрабатывает регистрацию нового пользователя:<br>- Проверяет наличие имени и хеша пароля<br>- Создает нового пользователя<br>- Вызывает user.registration()<br>- При успехе сохраняет пользователя и возвращает его данные |
| `socketLogin(data, socket)` | Обрабатывает вход пользователя:<br>- Проверяет данные для входа<br>- Создает временного пользователя для проверки<br>- При успехе сохраняет пользователя и возвращает его данные<br>- Временно генерирует тестовую карту 50x50 и запускает игру |
| `socketLogout(data, socket)` | Обрабатывает выход пользователя:<br>- Проверяет наличие токена<br>- Находит пользователя по guid<br>- Вызывает user.logout()<br>- Удаляет пользователя из хранилища |

# LobbyManager

## Назначение

Менеджер управляет игровыми лобби (комнатами): созданием, присоединением, выходом, исключением игроков, а также уведомляет всех клиентов об изменениях состояния лобби.

## Хранилища данных

| Хранилище | Описание |
| :--- | :--- |
| `this.users` | Хранит пользователей, привязанных к сокетам, где ключом выступает socket.id, а значением — объект пользователя |
| `this.lobbies` | Хранит все существующие лобби, где ключом является guid лобби, а значением — экземпляр класса Lobby |
| `this.userToRoom` | Обеспечивает связь пользователя с комнатой, в которой он находится. Ключ — user.guid, значение — lobby.guid |

## Методы-обработчики событий сокетов

| Метод | Описание |
| :--- | :--- |
| `handleCreateRoom` | Обрабатывает событие CREATE_ROOM. Создаёт новое лобби, добавляет создателя в комнату, подписывает сокет на эту комнату и уведомляет всех об обновлении списка комнат |
| `handleJoinRoom` | Обрабатывает событие JOIN_ROOM. Добавляет пользователя в существующее лобби, предварительно выходя из предыдущей комнаты, если пользователь уже был в какой-либо |
| `handleLeaveRoom` | Обрабатывает событие LEAVE_ROOM. Удаляет пользователя из текущего лобби и уведомляет его о выходе |
| `handleKickUser` | Обрабатывает событие DROP_FROM_ROOM. Позволяет создателю комнаты исключить другого пользователя из лобби |
| `handleGetRooms` | Обрабатывает событие GET_ROOMS. Возвращает клиенту список всех доступных лобби |
| `handleDisconnect` | Обрабатывает отключение клиента. Удаляет пользователя из его комнаты и очищает запись в хранилище users |

## Вспомогательные методы

| Метод | Описание |
| :--- | :--- |
| `sendError` | Отправляет клиенту сообщение об ошибке с указанным кодом через сокет |
| `getUser` | Возвращает объект пользователя по идентификатору сокета |
| `_leaveRoom` | Приватный метод, который удаляет пользователя из комнаты. Если пользователь был создателем комнаты или после удаления комната опустела, лобби уничтожается. В противном случае остальные участники получают уведомление об обновлении |
| `_destroyLobby` | Удаляет лобби из хранилища и уведомляет всех клиентов об обновлении списка комнат |
| `_getPublicRooms` | Формирует массив публичных данных всех существующих лобби для отправки клиентам |
| `_notifyRoomUpdate` | Отправляет обновлённое состояние конкретной комнаты всем её участникам через сокет |
| `_notifyRoomsListUpdated` | Рассылает обновлённый список всех доступных комнат всем подключённым клиентам |

# GameManager

## Назначение

Менеджер управляет игровым процессом, в частности экономической составляющей игры. Он создаёт экземпляры экономики для каждого пользователя, запускает игру и обрабатывает обновления состояния через колбэки.

## Хранилища данных

| Хранилище | Описание |
| :--- | :--- |
| `this.economies` | Хранит экземпляры экономики для каждого пользователя, где ключом выступает guid пользователя, а значением — экземпляр класса Economy |

## Методы-обработчики

| Метод | Описание |
| :--- | :--- |
| `callbackUpdate` | Метод обратного вызова, который вызывается при обновлении состояния экономики. Получает пользователя по guid через mediator, и если пользователь найден и авторизован, отправляет ему через сокет обновлённые данные с событием UPDATE_SCENE. Если пользователь не найден, отправляет сообщение об ошибке |
| `_createEconomy` | Приватный метод, создающий новый экземпляр экономики для пользователя. Принимает guid пользователя, стартовую точку и карту. Сохраняет созданный экземпляр в хранилище economies. При создании передаёт колбэк updated, который будет вызывать callbackUpdate при изменениях |

## Mediator взаимодействие

### Подписки на события

Менеджер подписывается на событие START_GAME через mediator. При получении этого события вызывается метод eventStartGame.

### Использование триггеров

В методе callbackUpdate используется триггер GET_USER_BY_GUID для получения пользователя по его идентификатору. Это позволяет получить данные о пользователе из UserManager без прямого обращения к нему.

## События

| Событие | Описание |
| :--- | :--- |
| `eventStartGame` | Обрабатывает событие START_GAME. Получает из данных guid пользователя, стартовую точку и карту. Через mediator запрашивает пользователя по guid. Если пользователь найден и авторизован, создаёт для него экземпляр экономики с помощью _createEconomy и отправляет клиенту подтверждение запуска игры с данными экономики. Если пользователь не найден, отправляет сообщение об ошибке |

# MapManager

## Назначение

Менеджер управляет игровыми картами: созданием карт, генерацией рельефа и ресурсов, обновлением юнитов и зданий, а также предоставлением информации о видимости для разных игроков в зависимости от их роли.

## Хранилища данных

| Хранилище | Описание |
| :--- | :--- |
| `this.maps` | Хранит все созданные игровые карты, где ключом является guid карты, а значением — экземпляр класса Map |

## Методы-обработчики сокетов

| Метод | Описание |
| :--- | :--- |
| `socketGenerateMap` | Обрабатывает событие GENERATE_MAP от клиента. Принимает параметры генерации (ширину, высоту, процент воды, гор, семя генерации, железо, нефть), создаёт новый экземпляр карты, генерирует рельеф и ресурсы, сохраняет карту в хранилище и отправляет клиенту результат сгенерированной карты |
| `socketGetMapParams` | Обрабатывает событие GET_MAP_PARAMS. Принимает guid карты, ищет карту по guid и возвращает клиенту параметры генерации этой карты |

## Mediator взаимодействие

### Подписки на события

Менеджер подписывается на событие START_GAME через mediator. При получении этого события вызывается метод eventStartGame.

### Триггеры (установленные обработчики)

| Триггер | Описание |
| :--- | :--- |
| `GET_RELIEF_HANDLER` | Обрабатывает запросы на получение рельефа карты. Принимает mapGuid и userGuid, проверяет существование карты и наличие роли пользователя, возвращает рельеф карты |
| `UPDATE_BUILDINGS_HANDLER` | Обрабатывает обновления зданий на карте. Принимает mapGuid, userGuid и массив зданий, проверяет права пользователя и обновляет каждое здание на карте |
| `UPDATE_UNITS_HANDLER` | Обрабатывает обновления юнитов на карте. Принимает mapGuid, userGuid и массив юнитов, проверяет права пользователя и обновляет каждого юнита на карте |
| `GET_VISIBILITY_HANDLER` | Обрабатывает запросы на получение видимых объектов на карте. Принимает mapGuid и userGuid, определяет роль пользователя, и если это не зритель, возвращает видимые сущности для данной роли |
| `GET_RESOURSE_VISIBILITY_HANDLER` | Обрабатывает запросы на получение видимых ресурсов. Работает аналогично GET_VISIBILITY_HANDLER, но возвращает видимые ресурсы для разведчиков |

## Вспомогательные методы

| Метод | Описание |
| :--- | :--- |
| `isUserInAnyLobby` | Проверяет через mediator, находится ли пользователь с указанным guid в каком-либо лобби |
| `getMapByGuid` | Ищет карту в хранилище maps по guid пользователя. Возвращает карту, если найдена, или undefined |
| `getRoleByGuid` | Определяет роль пользователя на карте. Принимает карту и guid пользователя, возвращает роль (например, peopleEconomy, mushroomArmy, spectator) или undefined, если пользователь не найден на карте |

## События

| Событие | Описание |
| :--- | :--- |
| `eventStartGame` | Обрабатывает событие START_GAME. Принимает объект с lobbyGuid и идентификаторами сервисов (peopleEconomy, peopleArmy, mushroomEconomy, mushroomArmy). Создаёт новую карту с переданными guid игроков, генерирует рельеф и ресурсы, сохраняет карту в хранилище. После этого отправляет HTTP-запросы на локальные сервисы для уведомления о начале игры (в коде закомментированы три из четырёх запросов) |

## Обработчики триггеров (Handlers)

| Обработчик | Описание |
| :--- | :--- |
| `getReliefHandler` | Получает рельеф карты. Проверяет существование карты и наличие роли пользователя, затем возвращает рельеф через метод map.getRelief() |
| `updateUnitsHandler` | Обновляет юнитов на карте. После проверок проходит по массиву юнитов и для каждого вызывает map.updateUnit с добавленной ролью пользователя |
| `updateBuildingsHandler` | Обновляет здания на карте. Работает аналогично updateUnitsHandler, но для зданий |
| `getVisibilityHandler` | Возвращает видимые сущности для роли пользователя. Проверяет, что пользователь не является зрителем, и возвращает результат map.getVisbileEntitiesByRole |
| `getResourseVisibilityHandler` | Возвращает видимые ресурсы для роли пользователя. Проверяет, что пользователь не является зрителем, и возвращает результат map.getVisbileSoursesByRole |
