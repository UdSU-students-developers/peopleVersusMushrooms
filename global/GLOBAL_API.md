# Описание API
Здесь описано API, используемое для lobby:


**Содержание**
1. Общие правила ответов
2. Основные события (emit -> on)
    * 2.1. createLobby (Создать лобби)
    * 2.2. joinToLobby (Присоединиться к лобби)
    * 2.3. leaveLobby (Выйти из лобби)
    * 2.4. dropFromLobby (Кикнуть игрока)
    * 2.5. startGame (Начать игру)
    * 2.6. setReady (Готовность игрока)
    * 2.7. getLobbies (Получить список лобби)
3. События, которые сервер шлет сам (on)
    * 3.1. lobbyUpdated
    * 3.2. lobbiesListUpdated
    * 3.3. lobbyDestroyed


# 1. Общие правила ответов
Все ответы приходят в формате, возвращаемом `this.answer.good(data)` или `this.answer.bad(errorCode)`.

**Успех:** { result: 'good', data: ... }

**Ошибка:** { result: 'error', code: number } список кодов ошибок см. [Здесь](../global/answerCodes.js).


# 2. Основные события (emit -> on)


## 2.1 createLobby (Создать лобби)
**Куда посылать:** `socket.emit(MESSAGES.CREATE_LOBBY, data)`

**Что посылать:**
```JavaScript
    {
        "guid": "string",         // GUID пользователя, создающего лобби
        "lobbyName": "string",    // Название комнаты
        "role": "any"             // Роль, которую занимает создатель (например, "player" или "spectator")
    }
```

Что возвращается:
**Успех:** Объект лобби `(метод lobby.get())`.

**Ошибка:** `1001` (Пользователь не найден).

**Доп. действия:** После успеха сервер сам разошлет всем событие `lobbiesListUpdated`.


## 2.2 joinToLobby (Присоединиться к лобби)
**Куда посылать:** `socket.emit(MESSAGES.JOIN_TO_LOBBY, data)`

**Что посылать:**
```JavaScript
    {
        "guid": "string",         // GUID пользователя, который заходит
        "lobbyGuid": "string",    // GUID лобби (равен GUID создателя)
        "role": "any"             // Выбранная роль
    }
```

Что возвращается:
**Успех:** Обновленный объект лобби.

**Ошибка:** `1001` (Пользователь не найден), `2003` (Лобби не существует), `2005` (Уже в этом лобби), `2004` (Лобби заполнено), `2017` (Не удалось добавить игрока).

**Доп. действия:** Рассылается `lobbyUpdated` всем в лобби и `lobbiesListUpdated` всем клиентам.


## 2.3 leaveLobby (Выйти из лобби)
**Куда посылать:** `socket.emit(MESSAGES.LEAVE_LOBBY, data)`

**Что посылать:**
```JavaScript
    {
        "guid": "string"          // GUID пользователя, который выходит
    }
```

Что возвращается:
**Успех:** `true` (data: true).

**Ошибка:** `1001` (Пользователь не найден), `2006` (Пользователь не в лобби).

**Важное поведение:** Если выходит создатель `(guid === lobby.lobbyGuid)`, лобби полностью уничтожается, а всем его участникам придет событие `lobbyDestroyed`.


## 2.4 dropFromLobby (Кикнуть игрока)
**Куда посылать:** `socket.emit(MESSAGES.DROP_FROM_LOBBY, data)`

**Что посылать:**
```JavaScript
    {
        "guid": "string",         // GUID создателя (кто кикает)
        "targetGuid": "string"    // GUID игрока (кого кикают)
    }
```

Что возвращается:
**Успех:** Обновленный объект лобби (без выгнанного игрока).

**Ошибка:** `242` (Нет данных), `1001` (Создатель не найден), `2016` (Цель не найдена), `2006` (Создатель не в лобби), `2010` (Только создатель может кикать), `2007` (Нельзя кикнуть себя), `2009` (Цель не в этом лобби).


## 2.5 startGame (Начать игру)
**Куда посылать:** `socket.emit(MESSAGES.START_GAME, data)`

**Что посылать:**
```JavaScript
    {
        "guid": "string"          // GUID создателя лобби
    }
```

Что возвращается:
**Успех:** `true` (data: true).

**Ошибка:** `1001` (Не найден), `2006` (Не в лобби), `2010` (Не создатель), `2012` (Не все игроки готовы).

**Важное поведение:** При успехе лобби уничтожается (игра передается другому модулю через `mediator.call`). Всем игрокам придет `lobbyDestroyed`.


## 2.6 setReady (Готовность игрока)
**Куда посылать:** `socket.emit(MESSAGES.SET_READY, data)`

**Что посылать:**
```JavaScript
{
    "guid": "string"          // GUID игрока
}
```

Что возвращается:
**Успех:** `true` (data: true).

**Ошибка:** `1001` (Пользователь не найден), `2006` (Не в лобби).

**Доп. действия:** Рассылается `lobbiesListUpdated`.


## 2.7 getLobbies (Получить список лобби)
**Куда посылать:** `socket.emit(MESSAGES.GET_LOBBIES, data)`

**Что посылать:**
```JavaScript
{
    "guid": "string"          // GUID запрашивающего (для проверки, что он юзер)
}
```

Что возвращается:
**Успех:** Массив объектов лобби `(Lobby.get() -> Array<Object>)`.

**Ошибка:** `1001` (Пользователь не найден).


# 3. События, которые сервер шлет сам (on) (p.s. хз, надо или нет, ну пусть будет)
Вам нужно подписаться на эти события сокета на клиенте:


## 3.1 lobbyUpdated
Приходит всем внутри конкретного лобби, когда кто-то зашел/вышел/стал ready/его кикнули.

**Data:** Обновленный объект текущего лобби.

## 3.2 lobbiesListUpdated
Приходит всем клиентам, когда меняется список лобби (создали новое, удалили старое, изменилось кол-во игроков).

**Data:** Массив всех лобби на сервере.

## 3.3 lobbyDestroyed
Приходит всем участникам лобби, когда создатель вышел или игра началась (лобби удалено).

**Data:** `{ lobbyGuid: "string" }`
