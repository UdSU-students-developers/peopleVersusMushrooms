# Рабочие заметки по mushroomsArmy

## Принципы разработки
- Обмен данными между серверными модулями выполнять только через `mediator`.
- Использовать БД минимально: основное состояние хранить в активных записях (in-memory).
- `T` — тип данных без методов (DTO), `I` — интерфейс поведения классов.
- Никакого `any`, `interface` для DTO, `@ts-ignore`.
- Целочисленные координаты (`Math.floor`) — дробные вызывают рассинхрон.
- Событийная модель «двойного действия»: удар/создание юнита = HTTP-вызов соседнему сервису + обработчик у себя.
- Карта — единственный источник истины о занятости клеток и ландшафте.

## Общая архитектура
- Backend: Node.js + TypeScript + Express + Socket.IO.
- Frontend: React + TypeScript + Socket.IO-client.
- База данных: SQLite3.
- Стилизация: CSS.

## Правила по mediator
- `Mediator` — единая шина обмена между модулями. Прямые вызовы методов запрещены.
- Все имена событий/триггеров хранятся в `config.ts`.
- Для каждого события фиксировать контракт payload (что приходит, что возвращается).
- Бизнес-логика — в модулях, медиатор только доставляет сообщения.
