Сущности

1.Термины и определения

Игрок — пользователь, вошедший в игру.

Юнит — управляемая игровая единица.

Спавн — место появления игрока на карте.

Ресурсы — различные игровые валюты.

Скорость роста — скорость роста травы на мицелии.

PvP — режим «игрок против игрока».

HP (Health Points) — показатель здоровья юнита, определяющий, сколько урона он может выдержать перед уничтожением.

DPS (Damage Per Second) — показатель урона в секунду, который юнит наносит атаками.

2.Цели и задачи
Цель

Разработать экономическую систему PvP-стратегии с элементами строительства базы.

Задачи

Реализовать клиентскую часть экономики.

Реализовать серверную часть экономики.

Создать сбалансированную систему строительства и развития базы.

Обеспечить стабильную работу мультиплеера.

Провести тестирование работы приложения.
Вход в игру 
Состоит из двух полей ввода:
1. Логин
2. Пароль(при вводе зашифровано точками)
3. чекбокс "запомнить меня"

И кнопок:
1. "Авторизоваться"
2. "зарегистрироваться"

### 3.2 Экран регистрации  
1. **Поле ввода "Логин"**
   - Валидация в реальном времени
   - Индикатор доступности логина

2. **Поле ввода "пароль"**  
   - при вводе зашифровано точками

3. **Поле ввода "Подтверждение пароля"**
   - Сравнение с исходным пароле
   - Визуальное подтверждение совпадения

4.   **Поле ввода "отображаемое имя"**

4. **Кнопка "Зарегистрироваться"**
   - Активируется только при выполнении всех требований

### требования логина и пароля
**Требования к логину:**
- Длина: от 5 до 20 символов
- Допустимые символы: латинские буквы (a-z, A-Z), цифры (0-9), символы подчеркивания (_)
- Не должен начинаться с цифры или символа подчеркивания
- Не должен содержать пробелов и специальных символов (@, #, $, %)
- Должен быть уникальным (проверка на занятость в реальном времени)
- не должен представлять из себя персональную информацию  

**Требования к паролю:**
- Минимальная длина: 8 символов
- Обязательное использование символов разных регистров (A-Z, a-z)
- Обязательное наличие хотя бы одной цифры (0-9)
- Рекомендуется использование специальных символов (!@#$%^&*)
- пароль не должен совпадать с логином 

**Дополнительные элементы:**
   - Ссылка "Создать аккаунт" для перехода к регистрации
   - Чекбокс "Запомнить меня" для сохранения сессии

   **забыли пароль:**
    - просит ввести логин

# UserManager

## Структура и назначение

UserManager расширяет BaseManager и управляет пользователями, их регистрацией, входом/выходом из системы.

## Основные компоненты

### 1. Конструктор

- Инициализирует хранилище пользователей `users` (ключ - guid, значение - экземпляр User)
- Настраивает Socket.IO обработчики событий:
  - `REGISTRATION` - регистрация
  - `LOGIN` - вход
  - `LOGOUT` - выход
  - `disconnect` - отключение сокета
- Настраивает mediator для получения пользователя по guid

## Пользователи

### Как устроены пользователи

Каждый пользователь — это отдельная запись со своими данными.

### Действия с пользователями

| Действие | Что происходит |
| :--- | :--- |
| Найти пользователя по его номеру | Возвращает данные пользователя, если он есть в системе и уже вошёл |
| Зарегистрировать нового | Приходят имя и секретный код. Создаётся запись о пользователе. Выполняются шаги для завершения регистрации. Если всё хорошо — пользователь сохраняется, и его данные отправляются обратно |
| Войти в систему | Проверяются присланные данные. Создаётся временная запись для проверки. Если всё верно — пользователь сохраняется, его данные возвращаются. Дополнительно временно создаётся простая тестовая картинка 50 на 50 и запускается игра |
| Выйти из системы | Проверяется, есть ли ключ доступа. Находится пользователь по его номеру. Выполняются шаги для выхода. Запись о пользователе удаляется |

## Управление комнатами (лобби)

### Зачем это нужно

Этот блок следит за игровыми комнатами: создание, вход, выход, исключение игроков, а также оповещает всех об изменениях в комнатах.

### Что хранится

| Что хранится | Описание |
| :--- | :--- |
| Связь: канал — пользователь | Хранит, какой пользователь связан с каким каналом связи (ключ — номер канала, значение — данные пользователя) |
| Список комнат | Хранит все существующие комнаты (ключ — номер комнаты, значение — все данные о ней) |
| Связь: пользователь — комната | Позволяет узнать, в какой комнате находится пользователь (ключ — номер пользователя, значение — номер комнаты) |

### Действия, которые происходят по сигналам

| Действие | Описание |
| :--- | :--- |
| Создать комнату | Приходит сигнал на создание. Создаётся новая комната, создатель добавляется в неё, подписывается на обновления этой комнаты, и всем рассылается свежий список комнат |
| Войти в комнату | Приходит сигнал на вход. Пользователь добавляется в выбранную комнату. Если он уже был где-то, то сначала выходит из той комнаты |
| Выйти из комнаты | Приходит сигнал на выход. Пользователь удаляется из текущей комнаты, ему приходит подтверждение |
| Исключить игрока | Приходит сигнал на исключение. Тот, кто создал комнату, может выкинуть другого игрока |
| Запросить список комнат | Приходит сигнал на получение списка. Отправляется перечень всех доступных комнат |
| Отключение клиента | Если игрок потерял связь, он удаляется из своей комнаты и стирается запись о нём |

### Вспомогательные действия

| Действие | Описание |
| :--- | :--- |
| Сообщить об ошибке | Отправляет игроку сообщение, что случилась ошибка, с указанием кода |
| Найти пользователя | Возвращает данные пользователя по номеру канала связи |
| Выйти из комнаты (внутреннее) | Скрытое действие, которое удаляет пользователя из комнаты. Если вышел создатель или комната опустела — комната удаляется полностью. Иначе оставшимся приходит уведомление об изменениях |
| Удалить комнату | Убирает комнату из общего списка и оповещает всех, что список комнат изменился |
| Подготовить список комнат для отправки | Собирает сокращённые данные обо всех комнатах, чтобы отправить их игрокам |
| Оповестить об изменениях в комнате | Отправляет всем, кто в комнате, её обновлённое состояние |
| Оповестить об изменении списка комнат | Рассылает всем подключённым игрокам обновлённый список всех комнат |

## Управление игрой

### Зачем это нужно

Этот блок отвечает за ход игры, а именно за её экономическую часть. Он создаёт отдельную экономику для каждого игрока, запускает игру и реагирует на изменения.

### Что хранится

| Что хранится | Описание |
| :--- | :--- |
| Экономика игроков | Хранит экономические данные для каждого игрока (ключ — номер игрока, значение — его экономика) |

### Действия

| Действие | Описание |
| :--- | :--- |
| Реакция на обновление | Это действие срабатывает, когда в экономике что-то меняется. Находит игрока по его номеру через посредника. Если игрок есть и он в системе — отправляет ему через канал связи свежие данные с пометкой «обновить сцену». Если игрока нет — сообщает об ошибке |
| Создать экономику (внутреннее) | Скрытое действие, которое создаёт новую экономику для игрока. Запоминает, где начальная точка и какая карта. Сохраняет эту экономику. При создании связывает её с действием «реакция на обновление», чтобы при любых изменениях оно срабатывало |

### Как общается с посредником

**На какие сигналы подписан**

Блок подписан на сигнал «ЗАПУСТИТЬ ИГРУ». Когда этот сигнал приходит, запускается действие «Начать игру».

**Какие сигналы сам подаёт**

В действии «реакция на обновление» он подаёт сигнал «НАЙТИ ПОЛЬЗОВАТЕЛЯ ПО НОМЕРУ», чтобы получить данные игрока, не обращаясь к тому блоку напрямую.

### Сигналы

| Сигнал | Описание |
| :--- | :--- |
| Начать игру | Срабатывает при сигнале «ЗАПУСТИТЬ ИГРУ». Из присланных данных берёт номер игрока, начальную точку и карту. Через посредника запрашивает данные игрока. Если игрок найден и вошёл в систему — создаёт для него экономику (скрытым действием) и отправляет игроку подтверждение запуска с экономическими данными. Если игрок не найден — сообщает об ошибке |

## Управление картами

### Зачем это нужно

Этот блок занимается игровыми картами: созданием, рисованием рельефа и размещением ресурсов, обновлением зданий и отрядов, а также показывает разным игрокам разную информацию в зависимости от их роли.

### Что хранится

| Что хранится | Описание |
| :--- | :--- |
| Список карт | Хранит все созданные карты (ключ — номер карты, значение — все данные о карте) |

### Действия по сигналам от игроков

| Действие | Описание |
| :--- | :--- |
| Создать карту по параметрам | Приходит сигнал на генерацию карты. Вместе с ним передаются размеры, количество воды, гор, зерно для случайности, наличие железа и нефти. Создаётся новая карта, на ней рисуется рельеф и раскладываются ресурсы. Карта сохраняется, и игроку отправляется результат |
| Получить параметры карты | Приходит сигнал с номером карты. Блок ищет эту карту и отправляет обратно её параметры (размеры, воду, горы и т.д.) |

### Как общается с посредником

**На какие сигналы подписан**

Блок подписан на сигнал «ЗАПУСТИТЬ ИГРУ». Когда он приходит, запускается действие «Начать игру».

**Какие сигналы обрабатывает**

| Сигнал | Описание |
| :--- | :--- |
| Запрос рельефа | Приходит с номерами карты и игрока. Проверяется, существует ли карта и есть ли у игрока роль на ней. Возвращается рельеф карты |
| Обновление зданий | Приходит с номерами карты и игрока, а также списком зданий. Проверяются права игрока, и каждое здание обновляется на карте |
| Обновление отрядов | То же самое, но для отрядов |
| Запрос видимых объектов | Приходит с номерами карты и игрока. Определяется роль игрока. Если он не зритель — возвращаются объекты, которые эта роль может видеть |
| Запрос видимых ресурсов | Работает так же, как запрос видимых объектов, но возвращает ресурсы, которые могут видеть разведчики |

### Вспомогательные действия

| Действие | Описание |
| :--- | :--- |
| Проверить, в комнате ли игрок | Спрашивает у посредника, находится ли игрок с указанным номером в какой-либо комнате |
| Найти карту по номеру | Ищет карту в общем списке. Возвращает карту, если нашёл, или ничего |
| Узнать роль игрока на карте | Определяет, какую роль (например, экономика людей, армия грибов, зритель) играет игрок на этой карте. Если игрока на карте нет — ничего не возвращает |

### Сигналы

| Сигнал | Описание |
| :--- | :--- |
| Начать игру | Срабатывает при сигнале «ЗАПУСТИТЬ ИГРУ». Получает номер комнаты и номера четырёх служб (экономика людей, армия людей, экономика грибов, армия грибов). Создаёт новую карту с этими номерами игроков, рисует рельеф и раскладывает ресурсы, сохраняет карту. Затем отправляет запросы на другие службы, чтобы уведомить их о старте игры (в коде три из четырёх запросов отключены) |

### Обработка запросов

| Обработчик | Описание |
| :--- | :--- |
| Отдать рельеф | Проверяет, что карта существует и у игрока есть роль, затем отдаёт рельеф |
| Обновить отряды | Проверяет права, затем проходит по списку отрядов и обновляет каждый, добавляя роль игрока |
| Обновить здания | То же самое, что с отрядами, но для зданий |
| Отдать видимые объекты | Проверяет, что игрок не зритель, и возвращает объекты, которые видит его роль |
| Отдать видимые ресурсы | Проверяет, что игрок не зритель, и возвращает ресурсы, которые видит его роль |
