## 1. Общее

### 1.1. Адрес сервера

`http://localhost:3001`

### 1.2. Используемый протокол

API полностью реализовано на http(s).

Формат возвращаемых значений `JSON`.

Все методы, если это особо не оговорено, имеют тип `POST` и параметры передают внутри своего тела.

## 2. Структуры данных

### 2.1. Общий формат ответа

`T` - какие-то данные. В случае успешного ответа возвращается `result = 'ok'` и поле `data` с данными.

В случае ошибки возвращается `result = 'error'` и поле `error` с кодом и текстом ошибки

```
Answer<T>: {
    result: 'ok' | 'error';
    data?: T;
    error?: {
        code: number;
        text: string;
    };
}
```

### 2.2 Lobby

Здесь и всюду далее под типом данных `lobby` будет иметься ввиду следующее
```
{
    lobbyName: string; - название лобби
    lobbyGuid: string; - гуид лобби
    playersGuids: {    - гуид каждого сервиса в лобби
        spectator:       string | null,
        peopleArmy:      string | null,
        peopleEconomy:   string | null,
        mushroomArmy:    string | null,
        mushroomEconomy: string | null,
    };  
    playerIsReady: {   - готовность каждого сервиса в лобби
        spectator:       true | false,
        peopleArmy:      true | false,
        peopleEconomy:   true | false,
        mushroomArmy:    true | false,
        mushroomEconomy: true | false,
    }; 
}
```

### 2.3 Unit

Здесь и всюду далее под типом данных `unit` будет иметься ввиду следующее
```
{
    x: number,         - целочисленная координата
    y: number,         - целочисленная координата
    type: string,      - тип/название юнита
    guid: string,      - гуид юнита
    visibility: number - видимость юнита
    role: string,      - роль сервиса, отвечающего за юнита
}
```

### 2.4 Building

Здесь и всюду далее под типом данных `building` будет иметься ввиду следующее
```
{
    x: number,         - целочисленная координата
    y: number,         - целочисленная координата
    type: string,      - тип/название здания
    guid: string,      - гуид здания
    visibility: number - видимость здания
    role: string,      - роль сервиса, отвечающего за здание
    size: number       - размер здания
}
```

### 2.5 Source

Здесь и всюду далее под типом данных `source` будет иметься ввиду следующее
```
{
    x: number,          - целочисленная координата
    y: number,          - целочисленная координата
    type: string,       - тип/название здания,
    saturation: number  - объем добываемого ресурса в тик
}
```

## 4. Подробно

### 4.1.1 GET_LOBBIES

Получение списка уже существующих лобби

**Параметры**

```
{
    guid: string; - гуид пользователя
}
```

**Успешный ответ**

```
Answer<[lobby]>
```


### 4.1.2 CREATE_LOBBY

Создание лобби

**Параметры**

```
{
    guid: string;      - гуид пользователя
    lobbyName: string; - название лобби
    role: string;      - роль отправителя 
}
```

**Успешный ответ**

```
Answer<lobby>
```

### 4.1.3 JOIN_TO_LOBBY

Присоединение к лобби

**Параметры**

```
{
    guid: string;      - гуид пользователя
    lobbyGuid: string; - гуид лобби
    role: string;      - роль отправителя 
}
```

**Успешный ответ**

```
Answer<lobby>
```

**Ошибки**

* `2003` - указанного лобби не существует
* `2004` - лобби заполнено
* `2005` - пользователь уже в этом лобби
* `2017` - роль пользователя уже занята

### 4.1.4 LEAVE_LOBBY

Покинуть текущее лобби

**Параметры**

```
{
    guid: string; - гуид пользователя 
}
```

**Успешный ответ**

```

```

**Ошибки**

* `2006` - Вы не в комнате

### 4.1.5 DROP_FROM_LOBBY

Покинуть текущее лобби

**Параметры**

```
{
    guid: string; - гуид пользователя,
    targetGuid: string; - гуид пользователя, которого выкидывают 
}
```

**Успешный ответ**

```

```

**Ошибки**

* `2006` - Вы не в комнате
* `2007` - Нельзя выгнать создателя комнаты
* `2009` - Игрок не в этой комнате
* `2010` - Игрок не владелец комнаты

### 4.2.4 UPDATE_UNITS

Обновить определенных юнитов на карте. 
- При отсутствии гуида на карте - добавление его на карту
- При наличие гуида и изменении положение - изменение
- При наличие гуида и неизменном положении - удаление

**Параметры**

```
{ 
    mapGuid: string, - гуид карты
    userGuid: string - гуид пользователя
    units: [
        {
            x: number,
            y: number,
            type: string,
            guid: string,
            visibility: number
        }
    ]
}
```

**Успешный ответ**

```
Answer<true>
```

**Ошибки**

* `3002` - Карты нет
* `3003` - Игрока нет на карте
* `3004` - Наблюдатель использует методы сервисов
