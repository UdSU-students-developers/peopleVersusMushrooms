***Медиатор: события и триггеры***

Событие (Event) — именованное уведомление о том, что в системе что-то произошло. Один источник вызывает уведомление (`call`), а все подписчики получают данные. Возвращаемого значения нет — это односторонняя рассылка «произошло такое-то событие».

API:
- `mediator.subscribe(name, func)` — подписаться на событие (добавить обработчик).
- `mediator.call(name, data)` — уведомить всех подписчиков, передать им `data`.

Пример: уведомление о регистрации пользователя

Источник (RegistrationManager) после успешной регистрации уведомляет всех:

```js
// В RegistrationManager.register() после сохранения в БД:
const user = { username, password, token };
this.mediator.call(this.EVENTS.USER_REGISTERED, user);
```

Подписчики (app.js или другие модули) реагируют без прямой связи с RegistrationManager:

```js
// В app.js — логирование
mediator.subscribe(mediator.EVENTS.USER_REGISTERED, (user) => {
    console.log(`Новый пользователь: ${user.username}`);
});

// Можно добавить ещё подписчиков в любом месте:
mediator.subscribe(mediator.EVENTS.USER_REGISTERED, (user) => {
    sendWelcomeEmail(user.username);
});
```

Итог: один вызов `call` — много независимых реакций, источник не знает, кто и сколько подписан.

---

Триггер (Trigger) — именованный запрос с ответом. Кто-то вызывает триггер (`get`) с данными и получает обратно результат от единственного зарегистрированного обработчика. Это паттерн «запрос–ответ» через медиатор.

API:
- `mediator.set(name, func)` — зарегистрировать обработчик (одна функция на имя).
- `mediator.get(name, data)` — вызвать обработчик с `data` и **вернуть** то, что он вернул.

**Пример: регистрация по имени триггера**

Менеджер при инициализации вешает обработчик:

```js
// В конструкторе RegistrationManager:
this.mediator.set(this.TRIGGERS.REGISTER, (data) => this.register(data?.username, data?.password));
```

Любой модуль (например, хендлер или другой сервис) может зарегистрировать пользователя **без импорта RegistrationManager**:

```js
// Где угодно, где есть только mediator:
const result = await mediator.get(mediator.TRIGGERS.REGISTER, { username: 'ivan', password: 'secret' });
if (result) {
    console.log('Зарегистрирован, токен:', result.token);
} else {
    console.log('Такой пользователь уже есть');
}
```

Итог: вызывающий код не знает про RegistrationManager — только про имя триггера `REGISTER` и формат данных/ответа.

---

Сравнение

|                    | Событие (Event)              | Триггер (Trigger)           |
|--------------------|-----------------------------|-----------------------------|
| Роль               | Уведомление «что-то случилось» | Запрос «сделай это и верни результат» |
| Подписчиков/обработчиков | Много (много подписок)   | Один (одна функция на имя)  |
| Возврат значения   | Нет                         | Да                          |
| Кто вызывает       | Тот, у кого произошло событие | Тот, кому нужен результат   |
