# peopleArmy

## Архитектура проекта

Проект состоит из **клиента** (React, `client/`) и **сервера** (Node.js + Express, `server/`). Ниже — устройство бэкенда и связи между файлами.

### Точка входа и конфигурация


| Файл                   | Назначение                                                                                                                                                |
| ---------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `**server/app.js`**    | Запуск приложения: создаёт DB, Mediator, менеджеры (например RegistrationManager), подключает роутер, слушает порт. При завершении (SIGINT) закрывает БД. |
| `**server/config.js**` | Единый источник настроек: имя сервера, порт, путь к БД, имена событий и триггеров медиатора.                                                              |


### Роутинг (HTTP → обработчики)


| Файл                                                               | Назначение                                                                                                                                                                               |
| ------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `**server/application/router/Router.js**`                          | Express Router: объявляет маршруты (например `GET /reg/:username/:password`) и привязывает к ним хендлеры. Получает менеджеры (например `registrationManager`) и передаёт их в хендлеры. |
| `**server/application/router/handlers/index.js**`                  | Реэкспорт всех хендлеров (useRegistrationHandler, notFoundHandler и т.д.).                                                                                                               |
| `**server/application/router/handlers/useRegistrationHandler.js**` | Обработчик регистрации: получает `registrationManager`, из `req.params` достаёт username и password, вызывает `registrationManager.register()`, отправляет JSON-ответ.                   |
| `**server/application/router/handlers/notFoundHandler.js**`        | Обработчик для любых неизвестных маршрутов (`*`).                                                                                                                                        |


Цепочка запроса: **запрос → Router → нужный handler → менеджер (при необходимости) → ответ**.

### База данных


| Файл                                       | Назначение                                                                                                           |
| ------------------------------------------ | -------------------------------------------------------------------------------------------------------------------- |
| `**server/application/modules/db/DB.js`**  | Подключение к SQLite (`data.db`), создание экземпляра ORM. Методы-обёртки для удобства (getUserByName, updateToken). |
| `**server/application/modules/db/ORM.js**` | Тонкий слой над БД: методы get, all, insert, update, delete для работы с таблицами без сырого SQL.                   |


Менеджеры получают экземпляр `DB` и обращаются к БД через `this.db.orm` или методы DB.

### Медиатор


| Файл                                                  | Назначение                                                                                                                    |
| ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `**server/application/modules/mediator/Mediator.js**` | Шина событий и триггеров: модули общаются по именам (из config), без прямых ссылок друг на друга. Подробнее — в разделе ниже. |


### Модули (бизнес-логика)


| Файл                                                                  | Назначение                                                                                                                                                                         |
| --------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `**server/application/modules/BaseManager.js**`                       | Базовый класс менеджеров: принимает `mediator` и `db`, сохраняет ссылки и копии EVENTS/TRIGGERS. От него наследуются доменные менеджеры.                                           |
| `**server/application/modules/exampleModule/RegistrationManager.js**` | Менеджер регистрации: наследует BaseManager, регистрирует триггер REGISTER и при успешной регистрации вызывает событие USER_REGISTERED. Работа с таблицей users через this.db.orm. |
| `**server/application/modules/exampleModule/Example.js**`             | Модель «активной записи» (пользователь/сессия): поля id, username, token; методы login, checkToken, get, getSelf, init. Может использоваться для логина и проверки токена.         |


### Вспомогательное


| Файл                                              | Назначение                                                                                    |
| ------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| `**server/application/Answer.js**`                | Формирование единого формата ответов (ошибки, успех с данными). Можно подключать в хендлерах. |
| `**server/application/modules/common/Common.js**` | Общие утилиты (например guid).                                                                |


### Связи между частями

```
config.js ─────────────────────────────────────────────────────────┐
    │                                                               │
    ▼                                                               ▼
app.js ──► DB ──► ORM (внутри DB)                    Mediator (EVENTS, TRIGGERS из config)
    │         │                                                               │
    │         └── data.db (файл БД)                                            │
    │                                                                          │
    ├──► RegistrationManager(mediator, db) ◄── подписывается на триггер REGISTER,
    │         │                                 вызывает событие USER_REGISTERED
    │         └── использует db.orm для users                                    │
    │                                                                          │
    ├──► Router(registrationManager) ──► useRegistrationHandler(registrationManager)
    │                                         │                                  │
    │                                         └── GET /reg/:username/:password   │
    │                                                                          │
    └──► mediator.subscribe(USER_REGISTERED, ...) ◄───────────────────────────────┘
```

Итог: **config** задаёт настройки; **app.js** собирает DB, Mediator, менеджеры и Router; **Router** связывает URL и хендлеры; хендлеры вызывают менеджеров; менеджеры используют **DB/ORM** и **Mediator** (события и триггеры).

## Медиатор: события и триггеры

Событие (Event) — именованное уведомление о том, что в системе что-то произошло. Один источник вызывает уведомление (`call`), а все подписчики получают данные. Возвращаемого значения нет — это односторонняя рассылка «произошло такое-то событие».

API:

- `mediator.subscribe(name, func)` — подписаться на событие (добавить обработчик).
- `mediator.call(name, data)` — уведомить всех подписчиков, передать им `data`.

Пример: уведомление о регистрации пользователя

Источник (RegistrationManager) после успешной регистрации уведомляет всех:

```js
// В RegistrationManager.register() после сохранения в БД:
const user = { username, password, token };
this.mediator.call(this.EVENTS.USER_REGISTERED, user);
```

Подписчики (app.js или другие модули) реагируют без прямой связи с RegistrationManager:

```js
// В app.js — логирование
mediator.subscribe(mediator.EVENTS.USER_REGISTERED, (user) => {
    console.log(`Новый пользователь: ${user.username}`);
});

// Можно добавить ещё подписчиков в любом месте:
mediator.subscribe(mediator.EVENTS.USER_REGISTERED, (user) => {
    sendWelcomeEmail(user.username);
});
```

Итог: один вызов `call` — много независимых реакций, источник не знает, кто и сколько подписан.

---

Триггер (Trigger) — именованный запрос с ответом. Кто-то вызывает триггер (`get`) с данными и получает обратно результат от единственного зарегистрированного обработчика. Это паттерн «запрос–ответ» через медиатор.

API:

- `mediator.set(name, func)` — зарегистрировать обработчик (одна функция на имя).
- `mediator.get(name, data)` — вызвать обработчик с `data` и **вернуть** то, что он вернул.

**Пример: регистрация по имени триггера**

Менеджер при инициализации вешает обработчик:

```js
// В конструкторе RegistrationManager:
this.mediator.set(this.TRIGGERS.REGISTER, (data) => this.register(data?.username, data?.password));
```

Любой модуль (например, хендлер или другой сервис) может зарегистрировать пользователя **без импорта RegistrationManager**:

```js
// Где угодно, где есть только mediator:
const result = await mediator.get(mediator.TRIGGERS.REGISTER, { username: 'ivan', password: 'secret' });
if (result) {
    console.log('Зарегистрирован, токен:', result.token);
} else {
    console.log('Такой пользователь уже есть');
}
```

Итог: вызывающий код не знает про RegistrationManager — только про имя триггера `REGISTER` и формат данных/ответа.

Сравнение


|                          | Событие (Event)                | Триггер (Trigger)                     |
| ------------------------ | ------------------------------ | ------------------------------------- |
| Роль                     | Уведомление «что-то случилось» | Запрос «сделай это и верни результат» |
| Подписчиков/обработчиков | Много (много подписок)         | Один (одна функция на имя)            |
| Возврат значения         | Нет                            | Да                                    |
| Кто вызывает             | Тот, у кого произошло событие  | Тот, кому нужен результат             |
