# Техническое задание: Модуль карты игры

## 1. Общие положения и требования
**Назначение:** Создание игровой карты, представляющей собой матрицу, содержащих данные о ландшафте, ресурсах и размещенных объектах.

**Размер:** Матрица 50x50 ячеек (базовый размер), с возможностью расширения.

**Состояние:** Карта является динамической — состояние ячеек (постройки, юниты) изменяется в ходе игры.

## 2. Генерация карты
Процесс генерации должен состоять из двух основных этапов: генерация ландшафта (проходимости) и наложение ресурсов.

### 2.1. Генерация ландшафта (проходимости)
**Метод:** Использовать алгоритм Simplex Noise.

**Цель:** Создание ландшафта, определяющего базовую проходимость клетки.

**Результат:** Для каждой ячейки устанавливается тип ландшафта:

- **0 (Равнины):** Проходимы для всех фракций.
- **1 (Вода):** Проходима только для Людей, непроходима для Грибов.
- **2 (Горы):** Проходимы только для Грибов, непроходимы для Людей.

**Хранение:** Матрица ландшафта хранится как отдельный двумерный массив размером N x N.

### 2.2. Генерация ресурсов (наполнение)
Ресурсы генерируются на втором этапе, поверх карты ландшафта, и ей подчиняются.

#### 2.2.1. Общие правила:
- **Запрет наложения:** В одной ячейке может находиться только один источник ресурса.
- **Минимальное количество:** Общее количество каждого типа ресурса на карте не может быть меньше заданного в настройках карты значения.
- **Размещение:** 
  - **Нефть и Железо** размещаются строго на **Равнинах (тип 0)**.
  - **Грибница** не является ресурсом, который генерируется на карте. Она создается игроком-грибом в начале игры и разрастается в процессе.
- **Возобновляемость:** Источники ресурсов делятся на виды:
  - **Нефть:** могут использовать только люди
  - **Железо:** могут использовать и люди, и грибы
- Все ресурсы возобновляемые.

#### 2.2.2. Паттерны размещения ресурсов
Генерация ресурсов должна происходить не случайным разбросом, а с использованием паттернов для создания логичных залежей:

- **Залежи:** Ресурсы (Нефть, Железо) генерируются плотными кластерами (кучами) диаметром от X до Y клеток, состоящими из связных клеток равнин.

## 3. Структура данных: Ячейка карты
Физически отдельной таблицы «Ячеек» не существует. Является активной записью.

Логически ячейка описывается:
- Ландшафт (0, 1, 2)
- Источник ресурса (null, нефть, железо) 
- Строение (null, id постройки)
- Юнит (null, id юнита)


## 4. Хранение данных
Вместо единой таблицы клеток используются отдельные словари/таблицы для сущностей.

### 4.1. Индексы для быстрого доступа
Для ответа на запрос get(x, y) необходимо иметь индексы по координатам для каждой из таблиц объектов.

## 5. Объекты на карте и их свойства

### 5.1. Постройка
**Поля:**
- id постройки
- Тип (Буровая, Шахта, Грибница, Казарма, Труба и т.д.)
- x, y
- health

### 5.2. Юнит
**Поля:**
- id юнита
- Тип
- x, y
- health

**Методы:**
- move(x, y): Проверка проходимости с учетом фракции и отсутствия других объектов.
- die(): Удаление юнита.

## 6. Целевые условия
Генерация карты должна способствовать выполнению основной цели игры:

**Цель:** **Уничтожение всех боевых юнитов вражеской фракции.**
